Mentor Recommendation

  • Construct feature vectors
    • Mentor - from document-topic distribution
    • Mentee - from preferences --> {'1': [0], '2': [1], '3': [1], '4': [1], '5': [0], '6': [0], '7': [0], '8': [0],​ '9': [0], '10': [0], '11': [0], '12': [0], '13': [0], '14': [0], '15': [0], '16': [0], '17': [0], '18': [0], '19': [0], '20': [0]}

  • To identify mentors similar to preference, there are 2 key decisions:

    • similarity measure - we use cosine similarity

    • similarity threshold - this is randomly chosen to be 0.5 but needs to be adjusted with evaluations of the results.

Similarity computation

  • We compute the cosine similarity between the mentor profiles and mentee profile

  •  Recommend the ones with similarity value greater than threshold

Recommendation evaluation

As the first part of evaluations we need to create ground truth:

  • Pick sample users, say 100

  • Recruit evaluators say 3 

  • Run one query (user preference) per evaluator.

  • For this query mark each sampled user as relevant (1) or not relevant (0).

    • Possibly done by looking up author's name on scholar websites to retrieve their interests and other profile details.

Evaluation

  • cosine similarity scores (CSS) can be used to fit a model on the probability of relevant value, for P(relevant)>0.5 we can use the corresponding cosine similarity threshold values.
